자료게시판






REQ-1 (FR/ 3) : 자료게시판 버튼위에 커서를 위치했을 때, 서브 메뉴를 보여줘야합니다.
REQ-2 (FR/ 2) : 각각의 서브 메뉴를 클릭하여 각 정보에 대응하는 창으로 이동해야합니다.
REQ-3 (FR/ 3) : 각 창에서는 보유하고 있는 자료, 새로운 자료를 생성하는 버튼, 기존에 있는 자료를 수정하는 버튼을 보여줘야합니다.
REQ-4 (FR/ 2) : 보유하고 있는 자료를 마우스로 클릭하면, 자료의 내용을 보여주는 창으로 넘어가야합니다.
REQ-5 (FR/ 1): 현재 자료에 대한 정보를 보여주는 시스템과, 뒤로가기 버튼을 누르면 이전의 창으로 돌아가는 시스템이 필요합니다.
REQ-6 (FR/ 2): 새로운 자료를 저장하는 버튼을 마우스로 클릭시, 자료를 저장하는 화면으로 이동해야합니다.
REQ-7(FR/ 1): 자료를 저장하는 화면에서는, 새로운 파일을 올리거나, 자료에 대한 정보를 작성하는 칸, 자료의 제목을 쓰는 칸, 업로드한 내용을 저장하는 버튼, 뒤로가기 버튼을 보여주어야 함
REQ-8(FR/3): 제목을 쓰는 칸을 마우스로 클릭하면, 키보드를 이용해 문자를 타이핑하는 것을 허용해야 하며, 정보를 작성하는 칸을 마우스로 클릭하면, 키보드를 이용해 문자를 타이핑 하는 것을 허용해야 하며, 저장 버튼을 누르면. 작성자 이름, 저장한 날짜를 보여주고 문서번호를 자동으로 저장하고. 이전 창으로 돌아가는걸 허용해야 합니다.
REQ-9(FR/5): 새로운 자료를 저장시 제목 우측에 N가 생성 된다.
REQ-10 (FR/ 2) : 수정 버튼을 누르면, 수정할 자료에 대한 편집 창으로 이동한다.
REQ-11(FR/ 1): 편집 창에서는 현재 자료를 삭제하는 버튼, 새로운 자료를 올리는 업로드 버튼, 글을 새로 쓰거나 삭제하는 칸, 제목칸, 현재 상태를 저장하는 버튼, 뒤로가기 버튼으로 보여주어야 합니다.
REQ-12(FR/3):저장 버튼을 누르면. 작성자 이름, 저장한 날짜를 보여주고 문서번호를 자동으로 저장하고. 이전 창으로 돌아가게 허용해야 합니다. 뒤로가기 버튼을 누르면 이전 창으로 돌아가게 하고, 글을 쓰는 칸을 클릭하여, 사용자가 키보드를 이용해, 타이핑을 지우거나 새로 작성하는것을 가능하게 하며, 제목 칸을 클릭시, 사용자가 키보드를 이용해, 제목을 변경하는 것이 가능하며, 삭제 버튼을 눌러 현재 자료를 삭제하게 허용해 주어야 합니다.
REQ-13(FR/5): 새로운 자료를 저장시 제목 우측에 N가 생성 된다.
REQ-14(FR/5): 관리자 권한을 가진 직원은 공지사항 버튼을 눌러, 공지사항 작성 창으로 이동 가능하다.
REQ-15(NFR/ 3) : 자료게시판의 자료는 가장 최신의 자료부터 위에서부터 보여준다. 각 자료 옆에는 수정 버튼이 존재하고, 새로운 자료 작성은 우측 하단에 위치한다. 새로운 자료 옆에는 N 문자가 표시됩니다. 
REQ-16(NFR/ 3) : 자료 업로드, 수정시 저장 버튼은 중앙 하단에 위치한다. 모든 돌아가기 버튼은 좌측 상단에 위치한다. 제목 칸은 중앙 상단에 위치하며, 글 작성칸은 중앙에 위치한다.
REQ-17(NFR/ 3) : 서브 메뉴는 회사가 원하는 순서로 보여준다.










결재게시판








REQ-1(FR/ 3) : 결재게시판 버튼에 거서를 위치했을 때, 결재 신청과 결재 승인, 진행중인 결재 버튼을 보여줍니다.
REQ-2(FR/3): 결재신청 버튼을 클릭할시,  결재신청 페이지로 이동해야합니다.
REQ-3(FR/2): 결재신청 페이지에서  요청할 상급자의 목록을 보여주고, 업로드 버튼 수정 버튼, 뒤로가기 버튼, 결재신청 버튼을 보여줍니다.
REQ-4(FR/1): 결재 창에서는, 결재할 서류를 업로드하는 버튼을 눌러, 사용자가 파일을 올리는 것을 허용해야 하며, 수정버튼을 눌러 업로드한 파일을 변경하는 것을 허용해 주어야 합니다.. 
REQ-5(FR/3): 결재 창에서 뒤로가기 버튼을 누르면, 현재 작성하던 내용을 저장하지 않고 이전 창으로 넘어갑니다. 결제 신청 버튼을 누르면, 현재 작성한 내용을 저장하고 이전 창으로 돌아가는 것을 허용해야 합니다.
REQ-6(FR/3) : 결재승인 버튼을 누르면, 결재 승인 창으로 이동합니다.
REQ-7(FR/2) : 결재 승인 창에서는, 사용자에게 온 결재를 보여줍니다.
REQ-8(FR/2): 각 결재를 클릭시 사용자는 결재 정보를 보는 창으로 이동합니다.
REQ-9(FR/1): 결재 정보 창에서는 결재서류, 승인 버튼, 반려 버튼, 뒤로가기 버튼을 보여줍니다.
REQ-10(FR/1): 승인 버튼을 클릭하면, 현재 결재를 승인하고 이전 창으로 이동합니다,
반려 버튼을 클릭하면, 현재 결재를 반려하고 이전 창으로 이동합니다.
뒤로가기 버튼을 클릭하면, 이전 창으로 이동합니다.
REQ-11(FR/3): 진행중인 결재 버튼을 클릭하면 진행중인 결재 페이지로 이동합니다.
REQ-12(FR/2): 진행중인 결재 페이지에서는 현재 사용자가 올린 결재나, 완료된 결재에 관한 정보를 보여줍니다.
REQ-13(FR/1): 각 결재를 클릭해, 해당 결재에 관한 정보를 보여주는 페이지로 이동하게 합니다. 결재 신청자의 직급과 신청서류의 내용 그리고 최종처리 일자를 확인 할 수 있어야 합니다.
REQ-14(FR/ 4): 결재 진행후 반려 혹은 승인 을 보여줘야 합니다.
REQ-15(FR/ 3): 처리할 결제가 있는 사용자는 메인 화면에서 처리할 결재서류 알림이 떠야합니다. 
REQ-16(FR/ 3):  결재서류 반려시 관리자의 피드백을 열람할수 있어야 합니다.
REQ-17(FR/3): 결제승인 창에서 처리서류와 미처리 서류의 갯수를 확인할수 있어야 합니다.
REQ-18(NFR/ 4) : 결재 알림은 메인 메뉴 프로필 사진 아래에 위치한다.
REQ-19(NFR/ 4): 결재승인, 반려, 진행중 세가지 상황을 캘린더 좌측에 위치한다.
REQ-20(NFR/ 4): 결재 신청 버튼은 결제페이지 하단 중앙에 위치하고, 업로드 버튼은 중앙에, 수정 버튼은 업로드 버튼 오른쪽에 위치한다.
REQ-21(NFR/ 4): 결재 정보 창에서 결재 서류를 가장 최신순부터 위에서 아래로 보여준다.










캘린더




REQ-1(FR/1): 캘린더창에는 현재 일정이 쓰여있는 달력과 편집 버튼을 보여준다.
REQ-2(FR/2): 달력을 누르면 달력을 보여주는 페이지로 넘어간다.
REQ-3(FR/2): 달력을 보여주는 페이지에서는, 현재 일정을 큰 달력에 넣어 보여준다.
REQ-4(FR/2): 편집 버튼을 누르면 달력 편집 페이지로 이동한다.
REQ-5(FR/2): 편집 페이지에서는 달력과 달을 바꿀 수 있는 화살표 적용범위 버튼, 날짜마다 글 작성 칸, 저장 버튼, 뒤로가기 버튼을 보여준다.
REQ-6(FR/1): 사용자는 화살표를 눌러 달력의 달을 바꾸는 것을 허용하며, 날짜를 눌러 해당 칸에 키보드로 일정을 작성 할 수 있게 허용하고, 저장 버튼을 눌러 현재 상황을 저장하고 메인페이지로 돌아가게 하거나, 뒤로가기 버튼을 눌러 메인 페이지로 돌아가게 한다.
REQ-7(FR/3): 사용자는 권한에 따라 적용 범위를 설정하여, 달력의 수정 내용을 다른 직원에게 공유하는 것이 가능하다.
REQ-8(FR/3): 최상위 관리자는 회사의 전반적인 일정을 입력하여 적용 범위를 설정하여, 회사 전직원의 캘린더에 등록하는 것이 허용된다.
REQ-9(FR/3): 중견급 관리자는 자신의 부서까지 적용 범위를 설정하여,달력의 수정 내용을 다른 직원의 캘린더에 공유하는 것이 허용된다.
REQ-10(FR/3): 일반 사원은 적용 범위를 자신의 팀, 또는 개인으로 설정하여 달력의 수정 내용을 공유하거나 공유하지 않는 것이 허용된다.
REQ-11(NFR/):  편집 페이지에서 달력은 화면 중앙에 위치하고, 각 작성 칸은 날짜별로 존재하며, 저장 버튼은 우측 하단, 적용 범위 설정 버튼은 우측 상단에 위치한다. 
REQ-12(NFR/4) : 캘린더는 메인 화면에서 자신의 프로필 아래에 위치하며, 
편집 버튼은 캘린더 우측 상단에 위치하고, 일정에 대한 내용은 달력 안 날짜 칸에 위치한다.
